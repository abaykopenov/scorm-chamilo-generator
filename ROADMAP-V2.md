# План безопасной модернизации (Roadmap v2.0)

Главный принцип этого плана — **"Не сломай то, что работает"**. Все новые функции будут добавляться как отдельные модули (параллельные ветки), которые можно включать и выключать, не затрагивая текущий, выстраданный нами генератор и V4 Pipeline.

---

## Фаза 1: Интерактивный AI-компоновщик (Human-in-the-Loop)
*Самая простая и эффективная доработка для UX. Пользователь сможет редактировать структуру курса ДО написания текстов.*

**Как внедрить безопасно:**
1. **Разделение API:** Сейчас фронтенд отправляет один большой запрос "Сделай всё". Мы разделим его на два эндпоинта: `/api/generate-outline` (только Фаза А) и `/api/generate-content` (Фазы Б и В). Текущий `/api/generate` не трогаем — он останется для функции "Генерировать в 1 клик".
2. **UI-редактор графа:** На фронтенде делаем красивое дерево (Drag-and-Drop) из JSON-структуры Оглавления, которое вернул `/api/generate-outline`.
3. **Хранилище состояния (Redux/Zustand):** Пользователь меняет заголовки, удаляет лишние пустые уроки. Обновленный JSON улетает на сервер в `/api/generate-content`, и наш V4 Pipeline послушно пишет тексты строго по отредактированному плану. Никаких рисков поломки парсера или RAG!

## Фаза 2: Архитектура LMS-Плагинов (Адаптеры)
*Цель: Подключить Moodle, iSpring и другие СДО, не ломая интеграцию с Chamilo.*

**Как внедрить безопасно:**
1. **Создание интерфейса (Interface):** В папке `lib/` создаем абстрактный класс `LmsAdapter`. У него будет всего два обязательных метода: `uploadTheory()` и `createNativeTest()`. 
2. **Изоляция Chamilo:** Текущий код из `chamilo-client.js` просто "оборачиваем" в этот адаптер (`ChamiloAdapter extends LmsAdapter`), ничего внутри не меняя.
3. **Новый адаптер (Например, Moodle):** Вы пишете `MoodleAdapter`, который выполняет те же два метода, но стучится в API Moodle.
4. **Переключатель на Front-End:** В интерфейсе появляется выпадающий список "Куда публиковать?". В зависимости от выбора бэкенд вызывает нужный адаптер. Логика генерации курса вообще не узнает о том, куда улетел пакет.

## Фаза 3: Multimodal RAG (Зрение для ИИ: таблицы и чертежи)
*Цель: Научить систему понимать картинки и схемы из исходного PDF.*

**Как внедрить безопасно:**
1. **Независимый парсинг графики:** Оставляем текущий парсер текста (`cleanRagText`) как есть. Пишем *параллельный* парсер, который ищет в PDF картинки и сохраняет их в папку `/tmp/images/`.
2. **Подключение Vision-модели (LLaVA / Moondream):** Только для найденных картинок запускаем локальную Vision-модель. Она смотрит на картинку и генерирует текст: *"На графике изображен рост продаж на 20%"*.
3. **Объединение в Qdrant:** Текст, сгенерированный Vision-моделью, отправляется в нашу текущую базу Qdrant как обычный текстовый чанк (просто с пометкой `type: image_description`).
4. **Бесшовность:** V4 Pipeline и Агент-Писатель даже не поймут, что они работают с картинками. Для них это будут обычные текстовые факты, из которых они смогут генерировать крутые слайды.

## Фаза 4: Плавный переход на xAPI (Tin Can)
*Цель: Дать продвинутую аналитику по каждому клику ученика, заменив устаревший SCORM 1.2.*

**Как внедрить безопасно:**
1. **Параллельный Экспортер:** Сейчас у нас есть папка или модуль `scorm/exporter` (который собирает `imsmanifest.xml`). Мы не трогаем его! Мы создаем рядом новый модуль `xapi/exporter`, который собирает ZIP-архив с файлом `tincan.xml` вместо манифеста.
2. **Инъекция JS-трекера:** В HTML-шаблоны слайдов добавляем маленький скрипт (слушатель событий). Когда ученик находится на слайде дольше 10 секунд, скрипт молча отправляет JSON (Statement) по стандарту xAPI: *"Ученик X прочитал слайд Y"*.
3. **Опция при загрузке:** При нажатии кнопки "Опубликовать", добавляем чекбокс `[ ] Использовать xAPI вместо SCORM`. Если включено — сервер прогоняет данные через новый упаковщик. Если выключено — работает старый железобетонный SCORM-архиватор. Возврата назад или поломок не будет.

## 🎯 С чего лучше начать?
Я рекомендую начать с **Фазы 1 (Интерактивный AI-компоновщик)**. Это самая быстрая задача с точки зрения программирования (в основном Frontend-работа на React/Vue), она ничем не рискует на сервере, но дает колоссальный "Wow-эффект" для конечного пользователя, который больше не будет чувствовать, что ИИ делает всё за "черной ширмой".
