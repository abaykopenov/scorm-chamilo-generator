# SCORM 1.2 Generator for Chamilo

Параметризуемый веб-сервис на Next.js для генерации SCORM 1.2 пакетов под Chamilo.

## Что умеет

- Создает черновик курса по брифу и параметрам структуры.
- Поддерживает иерархию `Course -> Module -> Section -> SCO -> Screen`.
- Позволяет менять структуру курса и параметры итогового теста.
- Экспортирует SCORM 1.2 ZIP с несколькими SCO и финальным тестовым SCO.
- Ограничивает количество попыток и время теста внутри самого пакета.
- Может генерировать курс через локальную LLM, например `Ollama`.
- Поддерживает RAG: загрузку материалов, индексацию в локальные векторы и учет контекста при генерации курса.
- Может сразу отправлять собранный SCORM ZIP в Chamilo через форму импорта.

## Локальный запуск

```bash
npm install
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000).

Если задан `TELEGRAM_BOT_TOKEN` (и `TELEGRAM_BOT_ENABLED` не отключен), вместе с сервисом автоматически запускается Telegram-бот.

## Telegram-бот (создание SCORM ZIP из чата)

1. Создайте бота через `@BotFather` и получите токен.
2. Скопируйте `.env.example` в `.env.local`.
3. Укажите минимум:

```bash
TELEGRAM_BOT_ENABLED=true
TELEGRAM_BOT_TOKEN=<ваш токен>
```

4. Запустите сервис:

```bash
npm install
npx prisma generate
npx prisma db push
npm run dev
```

При `/create` бот показывает **интерактивное меню настроек**:
- Количество модулей (◀️/▶️)

**Материалы (RAG):**

**История:**

**Администрирование** (только для `TELEGRAM_BOT_ADMIN_IDS`):

### Авторизация по email

Бот поддерживает авторизацию через корпоративный email:
1. Пользователь отправляет email (должен заканчиваться на домены из `ALLOWED_EMAIL_DOMAIN`).
2. Бот отправляет 6-значный код на почту (через SMTP).
3. Пользователь вводит код — получает доступ.

### Загрузка файлов

1. Отправьте документ в чат бота (`pdf/docx/doc/odt/rtf/txt/md/csv/json/html/xml`).
2. Бот скачает файл, сохранит и проиндексирует его.
3. Проиндексированные файлы автоматически используются в `/create` как RAG-контекст.
4. После загрузки появляются inline-кнопки: «Сгенерировать Курс», «Сгенерировать Тест», «Очистить материалы».

### Загрузка в Chamilo из бота

После генерации курса доступна inline-кнопка **«📤 Загрузить в Chamilo»**. Бот поочередно запросит:
- URL Chamilo портала
- Логин
- Пароль (не сохраняется)
- Код курса

### Rate Limiting

По умолчанию на каждого пользователя:
- **5 генераций в час** (`TELEGRAM_BOT_MAX_GENERATIONS_PER_HOUR`)
- **15 загрузок файлов в час** (`TELEGRAM_BOT_MAX_UPLOADS_PER_HOUR`)

## Локальная LLM

На экране создания курса выберите:

- `Шаблонный draft` для генерации без LLM
- `Ollama` для локального вызова `http://127.0.0.1:11434`
- `OpenAI-compatible` для локального сервера с `/chat/completions`

Пример для Ollama:

```bash
ollama serve
ollama pull qwen2.5:14b
```

Затем укажите в UI:

- `Provider`: `Ollama`
- `Base URL`: `http://127.0.0.1:11434`
- `Model`: `qwen2.5:14b`

Если LLM недоступна или ответит невалидным JSON, сервис сначала попробует промежуточный `line-plan` режим (модель формирует темы и тест по контексту книги в простом формате строк), и только потом применит extractive fallback.

## RAG-материалы (локально)

На экране создания курса можно:

1. загрузить материалы;
2. проиндексировать выбранные файлы;
3. включить RAG и выбрать `Top-K` чанков;
4. сгенерировать курс с учетом найденного контекста.

Что такое `Top-K чанков`:

- это количество самых релевантных фрагментов из материалов, которые передаются в модель;
- меньше значение = более узкий и точный контекст;
- больше значение = более широкий контекст, но выше риск добавить лишний текст.

Рекомендуемый старт для MVP: `5-8`.

Важно: если в RAG выбраны книги (`documentIds`), сервис ожидает реальную LLM-генерацию.  
Шаблонный режим `template` в таком сценарии вернет ошибку вместо fallback, чтобы не выдавать пустой шаблонный контент.

Чтобы книга реально влияла на содержание курса и вопросов теста:

- убедитесь, что статус книги `indexed`;
- используйте генеративную модель (не embedding);
- оставьте `RAG режим = Включен` и выберите книгу;
- начните с `Top-K = 5..8`, затем повышайте до `10..12`, если контекста не хватает.

MVP-индексация поддерживает текстовые форматы:

- `.pdf`
- `.docx`
- `.txt`
- `.md`
- `.csv`
- `.json`
- `.html` / `.htm`
- `.xml`

## Публикация в Chamilo

На странице курса заполните блок `Публикация в Chamilo`:

- `Portal URL`: адрес портала LMS
- `Username`
- `Password`
- `Course code`
- `Upload page path`
- `Login path`

По умолчанию используются пути:

- `Login path`: `/index.php`
- `Upload page path`: `/main/newscorm/lp_controller.php?action=upload`

Сервис:

1. логинится в Chamilo;
2. открывает страницу импорта SCORM;
3. отправляет ZIP напрямую.

Пароль не сохраняется в JSON курса.

## Qdrant + LangChain (optional)

This project now supports Qdrant retrieval through LangChain JS.

Environment variables:

- `QDRANT_URL` (example: `http://127.0.0.1:6333`)
- `QDRANT_COLLECTION` (default: `scorm_material_chunks`)
- `QDRANT_API_KEY` (optional)
- `QDRANT_ENABLED` (`true`/`false`, optional). If not set, Qdrant is enabled only when `QDRANT_URL` is set.

Quick local start:

```bash
docker run -p 6333:6333 qdrant/qdrant
```

If Qdrant is unavailable, indexing/retrieval falls back to local JSON vectors.

## Локальный переводчик (Argos Translate)

Сервис поддерживает автоматический машинный перевод сгенерированных курсов на другие языки (например, английский, казахский) с помощью локальной open-source модели `argostranslate` (без использования платных API).

### Установка (один раз)

```bash
# Создаем и активируем виртуальное окружение
python3 -m venv .venv
source .venv/bin/activate

# Устанавливаем пакеты (строго из бинарных сборок (wheels) во избежание ошибок с C++ компиляцией)
pip install argostranslate flask --only-binary=:all: --no-cache-dir
```

### Запуск микросервиса

В **отдельном терминале** (параллельно с Node.js/Telegram-ботом) запустите Python-сервер перевода:

```bash
# Возвращаемся в папку проекта:
cd scorm-chamilo-generator

# Включаем виртуальное окружение:
source .venv/bin/activate

# Запускаем скрипт:
python tools/translator/server.py
```

Сервер поднимется на порту `5005`. Необходимые языковые пакеты (словари) скачаются автоматически при первом долгом запросе на этот язык. Таким образом, в меню курса прямо в самом Telegram-боте (`/create`) вы сможете выбрать нужный язык, а остальную магию локального перевода сделает этот микросервис.
