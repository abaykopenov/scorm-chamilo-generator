# SCORM 1.2 Generator for Chamilo

Параметризуемый веб-сервис на Next.js для генерации SCORM 1.2 пакетов под Chamilo.

## Что умеет

- Создает черновик курса по брифу и параметрам структуры.
- Поддерживает иерархию `Course -> Module -> Section -> SCO -> Screen`.
- Позволяет менять структуру курса и параметры итогового теста.
- Экспортирует SCORM 1.2 ZIP с несколькими SCO и финальным тестовым SCO.
- Ограничивает количество попыток и время теста внутри самого пакета.
- Может генерировать курс через локальную LLM, например `Ollama`.
- Поддерживает RAG: загрузку материалов, индексацию в локальные векторы и учет контекста при генерации курса.
- Может сразу отправлять собранный SCORM ZIP в Chamilo через форму импорта.

## Локальный запуск

```bash
npm install
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000).

## Локальная LLM

На экране создания курса выберите:

- `Шаблонный draft` для генерации без LLM
- `Ollama` для локального вызова `http://127.0.0.1:11434`
- `OpenAI-compatible` для локального сервера с `/chat/completions`

Пример для Ollama:

```bash
ollama serve
ollama pull qwen2.5:14b
```

Затем укажите в UI:

- `Provider`: `Ollama`
- `Base URL`: `http://127.0.0.1:11434`
- `Model`: `qwen2.5:14b`

Если LLM недоступна или ответит невалидным JSON, сервис сначала попробует промежуточный `line-plan` режим (модель формирует темы и тест по контексту книги в простом формате строк), и только потом применит extractive fallback.

## RAG-материалы (локально)

На экране создания курса можно:

1. загрузить материалы;
2. проиндексировать выбранные файлы;
3. включить RAG и выбрать `Top-K` чанков;
4. сгенерировать курс с учетом найденного контекста.

Что такое `Top-K чанков`:

- это количество самых релевантных фрагментов из материалов, которые передаются в модель;
- меньше значение = более узкий и точный контекст;
- больше значение = более широкий контекст, но выше риск добавить лишний текст.

Рекомендуемый старт для MVP: `5-8`.

Важно: если в RAG выбраны книги (`documentIds`), сервис ожидает реальную LLM-генерацию.  
Шаблонный режим `template` в таком сценарии вернет ошибку вместо fallback, чтобы не выдавать пустой шаблонный контент.

Чтобы книга реально влияла на содержание курса и вопросов теста:

- убедитесь, что статус книги `indexed`;
- используйте генеративную модель (не embedding);
- оставьте `RAG режим = Включен` и выберите книгу;
- начните с `Top-K = 5..8`, затем повышайте до `10..12`, если контекста не хватает.

MVP-индексация поддерживает текстовые форматы:

- `.pdf`
- `.docx`
- `.txt`
- `.md`
- `.csv`
- `.json`
- `.html` / `.htm`
- `.xml`

## Публикация в Chamilo

На странице курса заполните блок `Публикация в Chamilo`:

- `Portal URL`: адрес портала LMS
- `Username`
- `Password`
- `Course code`
- `Upload page path`
- `Login path`

По умолчанию используются пути:

- `Login path`: `/index.php`
- `Upload page path`: `/main/newscorm/lp_controller.php?action=import`

Сервис:

1. логинится в Chamilo;
2. открывает страницу импорта SCORM;
3. отправляет ZIP напрямую.

Пароль не сохраняется в JSON курса.
