## 💭 Идеи на будущее

### Поддержка Moodle
- Интеграция через Moodle Web Services API (REST/JSON)
- Создание Quiz через `core_question_create_category` + `mod_quiz_add_question`
- Загрузка SCORM через `mod_scorm_insert_scorm_data_base`
- **Сложность:** Высокая (Moodle API более структурированный, но сложнее Chamilo)

### Мультиязычная генерация
- Параметр `outputLanguage` в промптах
- Автоопределение языка PDF через первые 500 символов
- Перевод курса на другой язык (промпт: "переведи на английский, сохрани структуру")

### Генерация из нескольких PDF
- Индексация нескольких документов в один RAG-коллекцию
- Генерация курса, комбинирующего факты из всех источников
- Указание приоритетов документов

### Dashboard аналитики
- Страница `/admin/dashboard`
- Статистика: курсов создано, среднее время, средний балл
- Графики: генерация по дням, распределение качества
- Лог ошибок LLM

### SCORM 2004 поддержка
- Поддержка sequencing & navigation
- Более продвинутая аналитика прохождения
- **Сложность:** Очень высокая (SCORM 2004 = сложный стандарт)

### Генерация изображений для слайдов
- Интеграция с Stable Diffusion / DALL-E
- Автоматическая генерация иллюстраций по тексту слайда
- Сохранение в SCORM-пакет

### Экспорт в другие форматы
- PDF (конспект курса)
- PowerPoint (PPTX)
- Markdown (для GitHub)
- xAPI (Experience API) для продвинутой аналитики

### Автоматическое тестирование качества
- E2E тесты: PDF → генерация → проверка структуры
- Smoke тесты: загрузка в Chamilo → прохождение теста → проверка балла
- CI/CD pipeline для автотестов

1. 🎤 Микросервис Аудио: Text-to-Speech (TTS)
Зачем: Чтобы каждый экран курса не только можно было прочитать, но и послушать (очень важно для инклюзивности и тех, кто учится с телефона в дороге). Как сделать: Поднять отдельный Python-сервис, например на базе VITS или Sylero TTS (работает локально, без интернета). Как это будет работать: После того как нейросеть сгенерировала текст экрана, Node.js кидает этот текст в сервис TTS. Тот переводит текст в .mp3 и Node.js вшивает этот аудиофайл на страницу SCORM-курса (кнопка Play ▶️).

2. 🎨 Микросервис Генерации Картинок (Stable Diffusion)
Зачем: Сейчас курсы, скорее всего, состоят из сплошного текста (мб с таблицами). Курсы выглядят суховато. Как сделать: Рядом с Ollama поднимается сервис ComfyUI или Stable Diffusion WebUI (через API). Как это будет работать: Нейросеть (Ollama) пишет текст, а затем сама придумывает промпт на английском (например: concept art of a robot arm assembling a car, clean style). Node.js отправляет это в сервис генерации картинок, получает готовый .jpg и вставляет в SCORM.

3. 🌐 Микросервис Автоматического Перевода (Machine Translation)
Зачем: Если компания огромная, то курсы нужно переводить на 3-5 языков. Делать это через Ollama очень долго (генерация текстов медленная). Как сделать: Использовать нейросети, специализированные только на переводах. Например, Argos Translate или NLLB (No Language Left Behind от Meta). Как это будет работать: Сгенерировали отличный курс на русском. Сразу же в фоне пнули микросервис, он за 10-15 секунд перегнал весь SCORM-пакет на казахский и английский, сохранив все HTML-теги и структуру.

4. 📊 Микросервис Аналитики и Мониторинга (Отдельно от Бота)
Зачем: Когда проектом начнут пользоваться 100 методистов, нужно понимать, кто какие модели использует, сколько времени уходит на курс, какая память на сервере. Как сделать: Поднять Prometheus + Grafana (вы, кстати, уже пытались запустить какой-то системный монитор agent.py/server.py — это шаги именно в эту сторону!). Как это будет работать: Node.js шлёт туда метрики ("Сгенерировали 10 курсов, среднее время 12 минут, нагрузка на Ollama 80%"). А вы в красивых графиках (Grafana) смотрите, справляется ли сервер или пора докупать железо.

5. 🕷 Микросервис-Скрапер (Internet Researcher)
Зачем: К вам пришёл пользователь, но у него нет PDF-документа, а тема новая (например, "Обзор законов Казахстана 2026 года"). База Ollama про это ничего не знает. Как сделать: Поднять сервис-паук (например на Python + Scrapy или Playwright), который по API ищет информацию в интернете, читает статьи, скачивает их и быстро отдаёт в вашу базу Qdrant. Как это будет работать: Вы кидаете в бота ссылку на статью из Википедии или новостной сайт, сервис её парсит, и бот сразу готов делать из неё курс.
