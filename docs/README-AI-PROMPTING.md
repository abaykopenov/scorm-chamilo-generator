# Идеальный Промпт: Руководство по созданию курсов через ИИ

Этот небольшой гайд поможет вам получать курсы самого высокого качества, выдавая правильные инструкции для нашего пайплайна локальных LLM (120B).

## Как ИИ читает ваши данные?

Каждое поле в форме генерации имеет свой "вес" и напрямую влияет на разные модули под капотом (Поиск по базе RAG, Пластичный Планировщик (Planner) и глубокий Генератор (Writer)). Ниже разобрано каждое поле.

### 1. Тема курса (Название)
* **Как влияет:** Это базовый поисковый запрос. С его помощью система ищет релевантные факты в загруженных PDF-книгах и документах через векторную базу Qdrant.
* **Как писать:** Максимально специфично. Избегайте общих слов. ИИ ищет совпадения по смыслу.
  * ❌ *Плохо:* "Основы", "Цифровизация", "Двойники"
  * ✅ *Отлично:* "Ключевые этапы построения цифрового двойника сложного промышленного объекта"
  * ✅ *Отлично:* "Как применять математическое моделирование для цифровой тени"

### 2. Цели обучения (Learning Goals)
* **Как влияет:** 
  1. Помогает поиску RAG цепляться за нужные темы.
  2. Определяет структуру курса. Каждая добавленная цель = один отдельный Модуль курса. ИИ подбирает факты под *конкретную* цель.
* **Как писать:** Цель должна быть действием, направленным на практику. Модель опирается на глаголы.
  * ❌ *Плохо:* "Понять всё", "Теория двойников".
  * ✅ *Отлично:* "Научиться отличать умную модель от цифрового мастера."
  * ✅ *Отлично:* "Понять процесс интеграции элементов двойника в производство."
* *💡 Секрет:* Если вы хотите курс из 3 Модулей, напишите ровно 3 конкретные цели. Система сама распределит их по модулям 1 к 1.

### 3. Длительность в минутах (Duration)
* **Точно ли минуты влияют на генерацию?** 
  * **ДА, но косвенно.** Система передает цифру минут в системные инструкции для Writer-модели. Если вы ставите 15 минут, LLM понимает, что нужно давать емкие выжимки. Если ставите 60 минут, LLM более склонна раскрывать темы детально, добавлять нюансы и не урезать абзацы.
  * Однако **Главный регулятор объема** — это физическая настройка Структуры (сколько Модулей, Секций и Экранов вы попросили). Если вы выберете 1 Модуль и 1 Экран, но поставите длительность "90 минут", ИИ просто напишет один большой текст, но физически это не займет у человека 90 минут. И наоборот.

### 4. Аудитория (Audience)
* **Как влияет:** Задает Tone of Voice (тон общения и стиль письма) для модели Writer.
* **Как писать:**
  * Если аудитория **"Новые сотрудники без технического бэкграунда"** — модель будет смягчать формулировки, разжевывать термины и приводить простые аналогии.
  * Если аудитория **"Инженеры-проектировщики"** — модель будет использовать строгую терминологию, дифференциальные уравнения (из вашего PDF) и не будет тратить время на "банальные" объяснения.

### 💡 Пример идеального запроса (Золотой стандарт)
Для получения идеального длинного текста без "воды" и без плейсхолдеров, используйте следующий формат:

* **Тема:** Архитектура и уровни зрелости цифрового двойника на производстве
* **Аудитория:** Инженеры среднего звена, переходящие на стандарты Индустрии 4.0.
* **Цели обучения:** 
  1. Понять разницу между цифровой тенью (SDS) и цифровым двойником. 
  2. Освоить этапы математического моделирования в производстве.
* **Минуты:** 45 минут (чтобы заставить LLM писать абзацы ~1000 символов).
* **Структура:** 2 модуля, 1 секция, 1 SCO, по 2 экрана.

При таких вводных система:
1. Вытащит из книги Прохорова и Боровкова идеальные куски.
2. Отдаст первую цель в Модуль 1, а вторую — в Модуль 2.
3. Writer-модель (gpt-oss:120b) напишет глубокие тексты без "отсебятины".

### Текущее состояние

Сейчас система **анализирует** изображения из PDF (через `lib/multimodal/vision-llm.js` + `image-extractor.js`), но **не генерирует** новые. Экраны курса получают заглушки — SVG-плейсхолдеры (из `lib/postprocess/screen-generation.js`).

### Варианты интеграции

#### 1. Stable Diffusion (локальный, рекомендуется)

**Модели:**
- **SDXL 1.0** — хорошее качество, ~7 GB VRAM
- **SD 3.5 Medium** — новее, лучше текст на картинках, ~12 GB VRAM
- **Flux.1 Schnell** — быстрая генерация, ~12 GB VRAM

**Серверы (API):**
- **ComfyUI** (рекомендуется) — `http://127.0.0.1:8188/api/prompt`
- **Automatic1111** — `http://127.0.0.1:7860/sdapi/v1/txt2img`
- **Forge UI** — форк A1111 с оптимизацией VRAM

**Env-переменные (планируемые):**
```env
# Stable Diffusion (ComfyUI)
IMAGE_GENERATION_ENABLED=true
IMAGE_GENERATION_PROVIDER=comfyui          # comfyui | automatic1111 | ollama | openai
IMAGE_GENERATION_BASE_URL=http://127.0.0.1:8188
IMAGE_GENERATION_MODEL=sdxl_base_1.0       # имя модели/checkpoint
IMAGE_GENERATION_STEPS=20                  # шаги генерации (качество vs скорость)
IMAGE_GENERATION_CFG_SCALE=7.0             # adherence to prompt
IMAGE_GENERATION_WIDTH=1024
IMAGE_GENERATION_HEIGHT=768
IMAGE_GENERATION_STYLE=educational         # educational | technical | diagram | photo
```

#### 2. Ollama (multimodal модели)

Некоторые модели поддерживают генерацию через Ollama:
```env
IMAGE_GENERATION_PROVIDER=ollama
IMAGE_GENERATION_BASE_URL=http://127.0.0.1:11434
IMAGE_GENERATION_MODEL=stable-diffusion    # или sdxl через ollama
```

#### 3. OpenAI DALL-E (облачный)

```env
IMAGE_GENERATION_PROVIDER=openai
IMAGE_GENERATION_API_KEY=sk-...
IMAGE_GENERATION_MODEL=dall-e-3
```

### Архитектура генерации (план)

```
Screen Title + Topic
       ↓
  LLM: "Сгенерируй prompt для иллюстрации по теме: {topic}"
       ↓
  Image Prompt (на английском)
       ↓
  Stable Diffusion API / DALL-E / Flux
       ↓
  PNG/WEBP файл → сохранить в SCORM assets/
       ↓
  block.type = "image", block.src = "assets/img_screen_1.webp"
```

### Установка Stable Diffusion (ComfyUI)

```bash
# 1. Клонировать ComfyUI
git clone https://github.com/comfyanonymous/ComfyUI.git
cd ComfyUI

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Скачать модель SDXL
# Положить в ComfyUI/models/checkpoints/
wget https://huggingface.co/stabilityai/stable-diffusion-xl-base-1.0/resolve/main/sd_xl_base_1.0.safetensors \
  -O models/checkpoints/sd_xl_base_1.0.safetensors

# 4. Запустить сервер
python main.py --listen 0.0.0.0 --port 8188
```
